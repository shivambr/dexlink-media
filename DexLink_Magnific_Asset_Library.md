# Dex-Link — Magnific Asset Library

> **How to use this file:**  
> Every image and video generated in Magnific has a permanent **identifier** (e.g. `rlaC22Jxtc`).  
> Token URLs in Magnific expire quickly, but identifiers NEVER expire.  
> To get a fresh download URL for any asset, call: `creations_wait(identifier)` → returns current `.url`  
> Always look up the identifier here before pushing to Buffer — never rely on saved token URLs.

## How Claude uses this file

When pushing any post to Buffer:
1. Look up the post date + topic in this file
2. Get the `Magnific ID`
3. Call `creations_wait([identifier])` → get the current `.url` field
4. Use that URL in the Buffer `create_post` assets field
5. After generating any new image/video, immediately add its identifier here

When generating new images:
- Always pass `rlahQq7xtc` as the DEX logo reference asset
- Add the new identifier to this file before the session ends
